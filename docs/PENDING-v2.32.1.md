## A. Config fixes still needed (NOT yet done) — `.claude-plugin/plugin.json`

All three are in the large **SessionStart** hook command (one long line).

### #2 — 4 routed reviewers are never synced to `~/.claude/agents/`

`skills/great_cto/SKILL.md` routes to these `subagent_type`s, but they are
absent from the `for AGENT in …` copy list in the SessionStart command, so they
are never written to `~/.claude/agents/great_cto-*.md` and dispatch silently
fails:

- `gdpr-reviewer`
- `us-privacy-reviewer`
- `dpdpa-reviewer`
- `digital-health-reviewer`  ← may already be handled by the in-flight
  digital-health feature work; verify before adding to avoid a duplicate.

**Fix:** append the missing names to the `for AGENT in …` list. Consider also
`coordinator` (referenced by the COORDINATE class in CLAUDE.md) — currently
unsynced too.

**Verify:** after a session start,
`ls ~/.claude/agents/great_cto-gdpr-reviewer.md` should exist.

### #4 — broken `apply-model-override.sh` path

The SessionStart command contains:

```
bash "\/scripts/apply-model-override.sh" 2>/dev/null || true
```

The path is `\/scripts/…` (stray backslash, missing `${PLUGIN_DIR}`), so the
file never resolves and `~/.great_cto/config: agent-model` is silently ignored.

**Fix:** change to
```
bash "${PLUGIN_DIR}/scripts/apply-model-override.sh" 2>/dev/null || true
```

**Verify:** set `agent-model: opus` in `~/.great_cto/config`, start a session,
confirm a `model: opus` rewrite happens on the copied agents.

### #6 — `&;` breaks SessionStart under bash/sh (portability)

The SessionStart command has six `&;` sequences (background a command, then a
stray `;`):

```
… 2>/dev/null &; fi      (×4)
… 2>/dev/null &; elif    (×2)
```

`zsh` accepts `&;` (so it works on macOS, the maintainer's shell — which is why
it has shipped fine since v1.0.139). `bash`/`sh` reject it with
`syntax error near unexpected token ';'`, which aborts the **entire**
SessionStart hook (no agent/command sync, no PROJECT/brain load, no status).
This breaks any user whose hook shell is bash/sh (many Linux, CI, Docker).

**Fix:** replace every `&;` with `&` (drop the stray semicolon). `… &; fi` →
`… & fi` and `… &; elif` → `… & elif` are valid in both bash and zsh.

**Verify:** extract the SessionStart command and run `bash -n` on it — must exit
0 with no syntax error. (Currently `bash -n` errors; `zsh -n` passes.)
